Powerful Features

Everything you need for real-time data processing and intelligent reactions

Incremental Processing

Process only affected results, not entire datasets. 10-100x faster than traditional query re-execution.

Real-time CDC

Capture database changes via Debezium CDC. Support for PostgreSQL, MongoDB, MySQL, and more.

Graph Queries

Write queries in Cypher. Match patterns across multiple sources with cross-source joins.

Multi-dimensional Cache

Fast result lookups with indexes by query, entity, and field value. Point-in-time queries supported.

AI-Powered Reactions

Integrate with LangChain4j and OpenAI. Trigger intelligent actions based on query results.

High Performance

Process 100,000+ events per second with <10ms p99 latency. Built on Project Reactor.

Get Started in Minutes

Quick setup with Docker Compose and Spring Boot 

# Start PostgreSQL with CDC enabled
docker-compose up -d postgres

# Initialize database
docker exec -i openrudder-postgres psql -U postgres -d orders < scripts/init-db.sql
# Build the project
mvn clean install

# Run the example application
cd openrudder-examples
export OPENAI_API_KEY=your-key
mvn spring-boot:run
// Define a continuous query
ContinuousQuery query = ContinuousQuery.builder()
    .id("ready-orders")
    .query("""
        MATCH (o:Order)
        WHERE o.status = 'READY_FOR_PICKUP'
          AND NOT EXISTS(o.driverAssigned)
        RETURN o.id, o.customer, o.location
        """)
    .sourceIds(Set.of(ordersSource.getId()))
    .build();
